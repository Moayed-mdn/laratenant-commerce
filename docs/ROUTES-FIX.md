# Routes Fix Report

The API routes configuration in `src/config/routes.ts` has been updated to match the actual Laravel backend routes. The `API_ROUTES` object was completely replaced with a corrected version that includes: proper auth endpoints (`csrfCookie`, `login`, `logout`, `me`, `register`) under `/api/v1/users/auth/`, restructured `dashboard`, `users`, `products`, and `orders` as functions returning nested objects instead of direct objects, and new endpoints for user management (`block`, `unblock`, `restore`), product restore, and order operations (`updateStatus`, `cancel`, `refund`). Additionally, `src/lib/api/auth.ts` was updated to use `API_ROUTES.auth.csrfCookie()` instead of a hardcoded `/sanctum/csrf-cookie` string, and `src/lib/api/orders.ts` was fixed to call `API_ROUTES.store(storeId).orders().updateStatus(orderId)` instead of the incorrect path. These changes ensure the frontend correctly communicates with the Laravel backend API.
